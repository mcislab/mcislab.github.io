  • Concern Mining

    Mining a software system to identify the concerns crosscutting it is tedious. Most of the existing tools only return a set of function names, cannot deal with noise in identifier names, and merely yield a set of words without any structure. We propose to use the system's development history to address these three shortcomings.

    Check out our ICSE 2010 paper »
